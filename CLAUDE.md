## Playwright screenshots / artifacts

When using the Playwright MCP tools (`browser_take_screenshot`, snapshot files, etc.), save outputs to `/tmp` rather than the repo. Pass an absolute path under `/tmp` for any `filename` parameter. The default `.playwright-mcp/` directory in the project root is gitignored, but writing to `/tmp` keeps the working tree clean and avoids accidental commits.

## Relationship to aoe4guides.com

We are a **sibling tool** to [aoe4guides.com](https://aoe4guides.com), not a fork or a competitor. Their site is the de-facto build-order publishing tool in the AoE4 community; we provide an editor/runner that imports from them so users can practice builds locally. Stay format-compatible â€” if aoe4guides drifts, we adapt.

- **Source code (MIT, theirs is open):** <https://github.com/jensbuehl/aoe4-guides>. Read this when you need to understand what HTML they emit, what asset paths they reference, or how their data is shaped. The `public/assets/pictures/` directory in that repo is the authoritative inventory of icons aoe4guides can put in a build description (categories: `resource/`, `abilities/`, `age/`, `civilization_flag/`, plus civ-specific `technology_<civ>/`, `unit_<civ>/`, `building_<civ>/`, `landmark_<civ>/`, `ability_<civ>/`).
- **Public API (no auth):**
  - `GET https://aoe4guides.com/api/builds/<id>` â€” single build payload (HTML descriptions, civ code, age-grouped steps).
  - `GET https://aoe4guides.com/api/builds?civ=<code>&orderBy=score&season=Season+<N>` â€” list endpoint, capped at 10 results. The `season` filter does work; `limit` does not.
- **Civ codes drift.** aoe4guides has quietly renamed several civ codes â€” verify with the API before adding a new one. Currently live codes vs. the older 2-letter forms we kept as legacy fallbacks: `KTE`/`KT` (Templar), `DRA`/`OOD` (Order of the Dragon), `GOH`/`GH` (Golden Horde), `SEN`/`SD` (Sengoku Daimyo). `ANY` means civ-agnostic â€” we map to `"unknown"` so the user picks manually.
- **Schema is loose and nullable.** aoe4guides emits `null` (not just missing) for empty optional fields like `author`, `map`, `gameplan`, `builders`. All zod schemas use `.nullish()` instead of `.optional()` for this reason.
- **Image-tag handling pipeline** (in `src/lib/importAoe4Guides.ts`'s `htmlToText`):
  1. `aoe4GuidesSrcToToken` (in `src/lib/aoe4GuidesIconMap.ts`) tries `AOE4GUIDES_ALIASES`, then `PATH_MIGRATION`, with a `.png`â†”`.webp` extension swap, plus civ-flag and civ-suffixed-villager fallbacks.
  2. `<img title="...">` / `alt="..."` extraction.
  3. Basename text fallback â€” any aoe4guides `<img>` that survives the prior steps becomes a capitalized text label from the filename.
  4. Plain-text `\bbuild\b` substitutes to `{{general/build.webp}}` (rally is always emitted as an `<img>`, so handled by step 1).
- **Don't hand-edit `src/data/generated/pathMigration.ts` or `icons.ts`** â€” they're regenerated by `scripts/sync-aoe4-data.ts` from aoe4world data. Aoe4guides-specific quirks (typos like `towara`â†’`tawara`, civ-flag rebases, etc.) belong in `AOE4GUIDES_ALIASES` in `src/lib/aoe4GuidesIconMap.ts`.
- **Coverage audit on demand:** `bun run scripts/check-aoe4guides-coverage.ts` fetches a curated build per civ from the live API and reports any `<img>` src that ends up unmapped or any `<img>` HTML surviving into note text. Run this after any aoe4guides outage / asset rename / when the smoke spec starts to fail.
- **Smoke E2E:** `tests/e2e/aoe4guides-smoke.spec.ts` (`bun run test:e2e:aoe4guides-smoke`) hits real aoe4guides â€” that's intentional, the suite exists to catch upstream drift. Excluded from the default `bun run test:e2e` via `--grep-invert @aoe4guides-smoke`.
